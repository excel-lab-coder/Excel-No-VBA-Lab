UIA（UI Automation）で他アプリへ自動入力 ― Selenium不要のRPA代替

💡 本編（VBAなし）との位置づけ

VBA禁止の完全閉域環境での代替手法は → QR Automation Lab 記事① およびQRシリーズ全編を参照してください。VBAが使えるなら、QRのような物理的なエミュレーションではなく、OSネイティブのAPIで確実な制御が可能です。

1. なぜ「UI Automation」なのか？

Internet Explorerの廃止に伴い、VBAから直接DOMを操作する「IE制御」は過去のものとなりました。現在Webスクレイピングや自動操作といえば「SeleniumBasic」が有名ですが、これには「ChromeDriverのバージョン管理」「環境構築（インストール）」が必要という、実務運用上の大きなハードルがあります。

そこで最強の選択肢となるのが、Windows OSに標準搭載されているアクセシビリティAPI「UI Automation (UIA)」です。インストール不要で、EdgeやChromeなどのモダンブラウザだけでなく、デスクトップアプリ（ERPや会計ソフト等）のボタンやテキストボックスまでVBAから直接制御できます。

2. 準備：UIAutomationClientの参照設定と「Inspect.exe」

VBAからUIAを扱うためには、まずVBE（Visual Basic Editor）のメニューから ツール ＞ 参照設定 を開き、「UIAutomationClient」にチェックを入れます。

さらに、操作したい画面の要素（ボタンの名前やID）を特定するために、Windows SDKに含まれるツール「Inspect.exe」や、オープンソースの「Accessibility Insights」を使用します。これを使って、クリックしたいボタンの AutomationId や Name プロパティを調べます。

3. 実践コード：Windows電卓を自動操作する

UIAの基本を理解するために、Windows標準の「電卓」アプリをVBAから起動し、「1 + 2 =」を自動でクリックするサンプルコードです。Webブラウザの操作も基本原理は同じです。

Sub UIA_Calculator_Automation() Dim uia As New CUIAutomation Dim root As IUIAutomationElement Dim cond As IUIAutomationCondition Dim calcWindow As IUIAutomationElement ' 1. 電卓を起動して少し待つ Call Shell("calc.exe", vbNormalFocus) Application.Wait Now + TimeValue("00:00:02") ' 2. デスクトップ全体（ルート要素）を取得 Set root = uia.GetRootElement ' 3. 電卓ウィンドウを Name プロパティで検索 Set cond = uia.CreatePropertyCondition(UIA_NamePropertyId, "電卓") Set calcWindow = root.FindFirst(TreeScope_Children, cond) If calcWindow Is Nothing Then MsgBox "電卓が見つかりません", vbExclamation Exit Sub End If ' 4. 操作用のローカル関数でボタンを順番にクリック ClickButton calcWindow, uia, "num1Button" ClickButton calcWindow, uia, "plusButton" ClickButton calcWindow, uia, "num2Button" ClickButton calcWindow, uia, "equalButton" MsgBox "UIAによる自動操作が完了しました。", vbInformation End Sub ' --- ボタンをクリックするためのヘルパー関数 --- Private Sub ClickButton(parent As IUIAutomationElement, uia As CUIAutomation, autoId As String) Dim cond As IUIAutomationCondition Dim btn As IUIAutomationElement Dim invPattern As IUIAutomationInvokePattern ' AutomationIdで要素を検索 Set cond = uia.CreatePropertyCondition(UIA_AutomationIdPropertyId, autoId) Set btn = parent.FindFirst(TreeScope_Subtree, cond) If Not btn Is Nothing Then ' Invokeパターン（クリック動作）を取得して実行 Set invPattern = btn.GetCurrentPattern(UIA_InvokePatternId) If Not invPattern Is Nothing Then invPattern.Invoke End If End If End Sub

4. Webブラウザ（Edge / Chrome）への応用

Webブラウザ内のフォームに入力する場合も、UIA_ValuePatternId を取得して SetValue メソッドで文字を流し込み、ボタンを Invoke するだけです。Seleniumのようなバージョン不整合に悩まされることなく、社内システムへの完全な自動入力を実現できます。

実務でのポイント： UIAは「画面に描画されている要素」を操作するAPIです。ブラウザのスクロールに隠れている要素は取得できないことがあるため、必要に応じてTabキー（SendKeys）と組み合わせるなどの工夫が実務では効果的です。

5. UIAの実務での落とし穴と対策

UIAは強力ですが、以下の点に注意が必要です：

  • タイミング問題：画面遷移後にすぐ要素を取得しようとすると Nothing が返る場合があります。Application.Wait や再試行ループを必ず実装してください。
  • 要素の特定が困難：Webアプリの場合、AutmationIdがなく Name も動的に変わるケースがあります。PropertyCondition を複数組み合わせた CreateAndCondition で絞り込みましょう。
  • 管理者権限の壁：管理者権限で起動しているアプリケーションの要素は、通常権限のExcel VBAからはアクセスできません。同じ権限レベルで実行する必要があります。
  • 仮想デスクトップ：Citrixなどの仮想デスクトップ環境では、UIAが正しく動作しないことがあります。その場合はQR Automation Labの手法が代替手段になります。
